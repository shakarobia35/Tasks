>  乐于钻研：面对复杂问题或新技能，他们享受深入学习和攻克难关的过程。
>  分享与创造：他们不仅消费内容，更热衷于动手创造（如编程、制作、写作）并分享成果。
> 社群归属：Geek文化具有社群属性，他们通过线上论坛、技术社区、线下活动等找到志同道合的伙伴。

### 何为Geek精神？

简单来说，Geek精神就是一种对特定领域展现出极致热情，并且用切实而深度的钻研态度落实这份热情的精神。***热爱驱动***、***极致钻研***、***拥抱创新***是他们的代名词。得益于艾伦·图灵、马克·扎克伯格、史蒂夫·乔布斯这些极具极客精神的人们一代代地突破自己、探索创新，我们的计算机领域延续着极客精神，进而能够以更为可观的迭代产物造福人类。

总而言之，Geek是一种因热爱而专注，因专注而专业，并乐于通过创造和分享来实现自我价值的精神状态。它代表的不是孤僻或怪异，而是一种充满激情、智慧和创造力的生活方式。

### GitHub是干什么的?

GitHub 是一个庞大的开源社区，它为人们搭建了一个可交互的平台，具有存储、共享、合作等功能。

依据官网的教程，我们能知道在GitHub里可以实现:

```
展示或共享代码；
随时间的推移跟踪和管理对代码的更改；
让其他人审查你的代码，并提出改进建议；
在共享的项目中开展协作；
寻找感兴趣的存储库、主题、代码、人员和组织，激发工作灵感，助力自己的学习。
```

> 有学长当年光是打开 GitHub 网站就花了半天，你有过类似经历吗？说说你的判断。

我并没有花很长时间就打开了 GitHub 网站。但是通过查阅资料和相关王也，我发现在2015年前后这种情况并不罕见。那时国内访问GitHub经常遭遇DNS污染、国际带宽拥堵，加上GitHub资源加载涉及大量CDN域名，如果不配置代理或修改hosts，确实可能一个页面转圈半天，甚至加载不全。即便现在，网络环境、运营商限制、本地DNS解析问题，或没有配置代理/镜像站，仍然可能导致访问速度极慢。

> PS:网上推荐的几种解决办法：
>
> 1. 在windows上，打开命令提示符，运行：ipconfig /flushdns 然后重新访问github；
> 2.  在windows上，点击设置>网络和Internet>代理，确保''自动检测装置''开启或手动代理关闭；
> 3.   网络换成个人热点。

### Fork和Pull request

fork,从原仓库复制建一个属于自己的分支（分叉)。此后所做的改动只在自己的分支上，不会影响到原仓库。

> *这就相当于，徒弟向师傅学艺，徒弟举一反三做出的创新都是源于自己的改动，不会影响到师傅*。(有意思的是，创建分支后会注明从哪里分支而来，就像在说"师承某某某")

此时，如果提交pull request 并被接受---师傅看到徒弟的创新并接受改变---师傅和徒弟的知识"仓库"就都会发生改变。

```
操作步骤:
1. 新建上传仓库
新建仓库，点击Code复制SSH地址，右键想存放文件的文件夹A并选择Git Bash，输入 git clone SSH地址(已生成子文件夹B)，打开文件夹B。
2. 用VSCode在readme文件里修改。右键文件夹B，打开Git Bash,输入git add .,git commit -m"v0.1",git push(推送到GitHub仓库中)。
3. 点击Pull request,说明修改原因，若合并修改，原仓库增加Readme文件，本人成为仓库贡献者。
```

##  学习笔记

### 基本知识

1. 创建存储库

   ```
   选择+，单击New repository,设置存储库名称及简介，设置"公   
   用"或"私用"，选择"添加README 文件"，创建存储库。
   ```

   > README 文件，即自述文件，是向用户和开发者清晰介绍项目基本信  息、使用方法和相关指南的“使用说明书”、“第一印象” 。通常采用Markdown格式。
2. 创建分支

   默认情况下，存储库有一个名为 main 的分支，它被视为最终分支。通过分支可以同时拥有不同版本的存储库。而在合并主分支之前，不同分支上完成的工作不会显示在主分支上。
   ```
   单击codes,点击main下拉菜单，输入分支名称，单击"创建分支”(从main创建新分支)

3. 编辑和提交

   ```
   在新建的分支下单击README.md，单击编辑按钮，编辑完成后单击"提交更改"，描述更改的提交信息，单击"提交更改"。
   ```

4. 拉取请求(Pull Request)

   打开拉取请求后，可以提出更改，要求某人审查和提取您的贡献并将其合并到其分支中。 拉取请求显示两个分支中内容的差异。发生的变化、增减会以不同的颜色显示。

   ```
   单击pull request，点击新建拉取请求，选择需要比较的main和分支，创建拉取请求。
   ```

5. 合并拉取请求

   将分支上的更改合并到main分支中。

   ```
   点击完创建拉取请求之后会弹出新页面，点击合并拉取请求，点击确认合并，点击删除分支。
   ```

6. 下载和上传文件

   导航，单击Code,下载。Codes,add files,拖拽上传。

   > [个人认为比较重要的表格](https://docs.github.com/zh/get-started/start-your-journey/downloading-files-from-github#understanding-the-differences-between-downloading-cloning-and-forking)
