# 页面编后感

这次编写网页，我收获很多。想要把心里的想法具现为真正的网页的确不是一件容易的事情。

## 图层覆盖

我的原始想法很简单：页面上方做一个视差滚动特效，下面做一个图片画廊，点击图片可以跳转到详情页。但是，当我把两段代码拼在一起时，却发现它们叠在了一起——画廊区域跑到了视差区域的“背后”，滚动时根本看不到。

我借助AI，意识到视差区域的元素使用了绝对定位，导致它脱离了正常的文档流，后续的块级元素会被覆盖。解决思路就是给视差区域一个明确的高度（例如 100vh），让它独占一屏，后续内容自然就会排在它的下方。同时，要确保视差区域的 overflow: hidden 不会把多余内容切掉。

## 插入 Markdown 文档

我想把自己用 Typora 写的一篇笔记也放进网页里,于是采用“导出 HTML 再嵌入”的方法：

1. 在 Typora 中将 one.md 导出为 one.html。
2. 复制` <body> `内的所有 HTML 内容。
3. 粘贴到主页面中视差区域和画廊区域之间。
4. 同时复制 Typora 生成的 CSS 样式到主页面的 `<style>` 中，保留原文排版。


收获：学会了在静态页面中复用外部文档内容，理解了 HTML 和 CSS 的模块化思想。虽然每次修改 Markdown 都需要重新导出并粘贴，但对于不频繁更新的内容来说，这已经足够简单可靠。

四、将项目托管到 GitHub

最后，我把整个项目上传到了 GitHub 仓库 shakarobia35/my-website。通过截图中提示的命令行操作，我依次执行了：

```bash
git init
git add README.md
git commit -m "首次提交"
git branch -M main
git remote add origin https://github.com/shakarobia35/my-website.git
git push -u origin main
```



并添加了 README 文件。
